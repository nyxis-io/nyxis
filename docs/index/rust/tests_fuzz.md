# e2e.rs

DOES: End-to-end integration tests that drive `nxs-import` and `nxs-export` binaries as child processes, verifying JSON and CSV roundtrips produce value-equivalent output and that a 10k-record import completes under 1 second on release builds.
SYMBOLS:
- e2e_json_roundtrip_value_equivalent() -> ()
- e2e_csv_roundtrip_value_equivalent() -> ()
- e2e_10k_records_under_threshold() -> ()
DEPENDS: assert_cmd, tempfile, serde_json
PATTERNS: subprocess-integration-test, roundtrip-equality, perf-smoke-gate
USE WHEN: Verifying end-to-end correctness and advisory performance of the converter binaries together; use `exit_codes.rs` for error-path coverage.

- export_exits_2_when_unknown_format() -> ()
- export_exits_5_when_input_file_missing() -> ()
- export_exits_3_when_nxb_bad_magic() -> ()
- inspect_exits_2_when_no_input() -> ()
- inspect_exits_5_when_input_file_missing() -> ()
- inspect_exits_3_when_nxb_bad_magic() -> ()
DEPENDS: assert_cmd, predicates, tempfile
PATTERNS: subprocess-integration-test, exit-code-matrix
USE WHEN: Verifying the error-exit-code contract for all three converter CLIs; complement to `e2e.rs` which covers the happy path.

# fuzz_decode.rs

DOES: Minimal libFuzzer target that feeds arbitrary bytes directly into `nxs::decoder::decode`, ensuring the decoder never panics or triggers undefined behaviour on malformed input.
SYMBOLS:
- fuzz_target (macro entry point)
DEPENDS: libfuzzer_sys, nxs::decoder
PATTERNS: coverage-guided-fuzzing, panic-freedom
USE WHEN: Fuzzing the decoder read path against arbitrary byte sequences; use `fuzz_writer_roundtrip.rs` to also exercise the writer.

# fuzz_target_1.rs

DOES: Placeholder libFuzzer target generated by `cargo fuzz init`; the fuzz body is empty and does nothing with the input bytes.
SYMBOLS:
- fuzz_target (macro entry point, no-op body)
DEPENDS: libfuzzer_sys
PATTERNS: scaffold-placeholder
USE WHEN: Never use directly; exists only as the default cargo-fuzz scaffold. Prefer `fuzz_decode.rs` or `fuzz_writer_roundtrip.rs` for actual fuzzing.

# fuzz_writer_roundtrip.rs

DOES: libFuzzer target that interprets fuzz input bytes as a synthetic schema (variable key count) and record sequence, writes them through `NxsWriter`, then verifies the resulting `.nxb` bytes decode without panicking, confirming write→decode roundtrip safety.
SYMBOLS:
- fuzz_target (macro entry point)
DEPENDS: libfuzzer_sys, nxs::writer, nxs::decoder
PATTERNS: coverage-guided-fuzzing, roundtrip-panic-freedom, schema-driven-generation
USE WHEN: Fuzzing the full writer-to-decoder pipeline; use `fuzz_decode.rs` instead when you only want to stress the decoder against arbitrary bytes without involving the writer.

- import_json_stdin_spills_to_tempfile_and_cleans_up() -> ()

- import_json_with_schema_hint_skips_inference() -> ()
DEPENDS: assert_cmd, predicates, tempfile
PATTERNS: subprocess-integration-test, stdin-spill-verify, schema-hint-single-pass
USE WHEN: Testing the JSON import path specifically (stdin, spill, schema hint); use `e2e.rs` for full import→export roundtrips.
