# AGENTS.md

Guidance for maintainers, contributors, and automated assistants working in this repository.

## About

Nyxis (`nxs`) is a zero-copy, bi-modal serialization format: human-readable `.nxs` source compiled to binary `.nxb`. This repo is the BSL-1.1 open core: spec, Rust compiler and reader, conformance vectors, benchmarks, browser demos, and MCP server.

| Repo | Role | Local path |
| --- | --- | --- |
| `nyxis` | Spec, Rust core, conformance, benchmarks, site, MCP | `nyxis/` |
| `nyxis-drivers` | MIT language SDKs | `../nyxis-drivers` |
| `nyxis-extensions` | Private commercial crates under EULA | `../nyxis-extensions` |

Ground truth:

- `SPEC.md` is the wire-format contract.
- `NXS_v1.3.md` is the compact-encoding design doc (release train, driver tiers, measured numbers).
- `COMMERCIAL.md` defines licensing boundaries.
- `rust/src/query.rs` contains the zero-copy Reader API used by extensions.
- `docs/index/_root.md` is the LOI navigation index for task-oriented code lookup. Some language-driver entries are stale because those implementations now live in `../nyxis-drivers`.

## Repository Layout

```text
rust/         # reference compiler, writer, reader, CLIs, WAL, tests
mcp/          # Go MCP server (nxs-mcp)
conformance/  # golden vectors and language runners
bench/        # cross-format benchmark harness
site/         # Vue marketing site, browser demos, bench fixtures
examples/     # sample .nxs sources
docs/index/   # LOI navigation index
```

Language implementations are not owned here. Edit drivers in `../nyxis-drivers`.

## Build, Test, Lint

Run from `nyxis/` unless noted.

```bash
# Rust core
make lint-rust          # cargo fmt --check + clippy -D warnings
make fix-rust           # cargo fmt
make test-rust          # cargo test --release
make test-rust-ci       # test-rust + compile examples/
make convert-test       # converter integration tests

# MCP server (Go)
make lint-mcp           # gofmt check + go vet
make fix-mcp            # gofmt -w
make build-mcp          # builds bin/nxs-mcp
make test-mcp           # go test ./...

# Conformance
make conformance        # generate vectors + run language runners
make conformance-run-js # run one language runner
make conformance-prefetch PREFETCH=1

# Fuzzing (nightly required)
make fuzz               # 60s; override with FUZZ_TIME=300 make fuzz
```

Default targets:

- `make test` runs Rust core tests only.
- `make lint` runs `lint-rust` and `lint-mcp`; this is also the pre-commit hook.
- `make install-git-hooks` sets `core.hooksPath` to `.githooks`.
- Bypass a single hook run with `SKIP_HOOKS=1 git commit ...`.

## CLI Syntax

The `nxs` binary uses subcommands:

```bash
cd rust && cargo build --release
./target/release/nxs compile ../examples/user_profile.nxs
./target/release/nxs compile input.nxs output.nxb
```

## Driver Tests

Do not rely on this repo's `make test-all`, `make lint-all`, or aggregate `make fix` for Python, Go, C, Swift, Kotlin, or C#. Several targets still reference removed in-repo language directories. Use the sibling driver repo instead:

```bash
make -C ../nyxis-drivers test
make -C ../nyxis-drivers lint
make -C ../nyxis-drivers fix
```

The core Makefile can auto-clone `../nyxis-drivers` for JS-dependent SDK and demo targets when missing:

```bash
git clone --depth 1 https://github.com/nyxis-io/nyxis-drivers.git ../nyxis-drivers
```

## Fixtures And Conformance

Bench/demo fixtures and conformance vectors are separate.

| Path | Used for |
| --- | --- |
| `site/bench/fixtures/` | Default `make fixtures` output in this repo; browser demos and `site/bench/` harness |
| `bench/fixtures/` | Default fixture path when generated through `nyxis-drivers` |
| `conformance/` | Golden conformance vectors generated by `make conformance` |

`make conformance` runs `conformance-generate` first and does not require `make fixtures`. It does require `../nyxis-drivers` for the Go, C, Swift, Kotlin, and C# runners.


All other warnings are errors (`-D warnings`).

## Binary Format Conventions

- Magic bytes: `NYXB` (file), `NYXO` (object), `NYXL` (list or columnar layout).
- `.nxb` is the compiled binary format; `.nxs` is the human-readable source format.
- MIME types: `application/nxb`, `application/nxs`.
- All integers are little-endian.
- `$` (Keyword) values encode as a `uint16_t` dictionary index. `get_keyword` is not implemented in the Rust reader yet and returns `Err(NxsError::UnsupportedFieldType)`.

## Agent Constraints

- Keep extension-only behavior, proprietary implementation details, and license material out of this public repo.
- `../nyxis-extensions` depends on this crate through the path dependency `../nyxis/rust`; do not publish extension dependencies from here.
- Match existing style, bounds-check offsets, and avoid silent out-of-bounds reads.
- If a subdirectory needs additional instructions, prefer an agent-neutral `AGENTS.md` scoped to that directory.
