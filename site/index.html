Nyxis is an infrastructure serialization standard — not application software. Human-readable .nxs compiles to memory-mapped .nxb you query by tail-index, without JSON.parse() or full-file decode passes.

Row layout: 7 µs time-to-first-record on EPYC 9R14; 37 µs on Linux inotify (streaming, P50).

Columnar layout: 1.3× Arrow IPC on AMD EPYC 9R14 AVX-512; at parity on Apple Silicon.

PAX layout: mixed scroll + column scan in one file (v1.2; page streaming per SPEC §4.5). v1.2 supports row, columnar, and PAX layouts with streamable sealing: emit schema and records incrementally, then seal the tail-index when the segment is complete.

Bimodal wire format

user {
  id = 42
  name "ada_lovelace"
  score ~ 98.6
  active ? true
}

Sigil-typed .nxs compiles to aligned .nxb cells. Readers mmap the wire image, use the tail-index for record offsets, and decode fields by key — no full-file parse pass.

Five pillars

  • Fast 8-byte aligned atomic cells; zero-copy reads without deserialization.
  • Flexible LEB128 bitmask per record; sparse objects pay nothing for absent fields.
  • Compressible Interned field dictionary; records store 2-byte indices, not repeated strings.
  • Human readable .nxs is self-describing plain text — the source is the schema.
  • Streamable Writers stream records before the tail-index exists; append-only WAL mode seals to indexed .nxb on demand.
  • AI-native The included MCP server exposes NXS files as typed tools for AI agents — the only binary serialization format with native AI agent access.

Benchmarks

Performance claims on this page are backed by a public benchmark suite covering five workloads (A–E) against Protobuf, FlatBuffers, Cap'n Proto, and Apache Arrow — across three platforms: macOS Apple Silicon, Linux Haswell, and AWS EPYC 9R14 (AVX-512). Results include honest losses: columnar files are ~15% larger than Arrow IPC at 1M records; row-oriented cold open trails FlatBuffers and Cap'n Proto on small files; dense analytics on row layout are 112× slower than Arrow — use columnar or the Arrow bridge; PAX col scan requires page_size ≥ 32,768 to beat row layout on x86 DRAM. Full methodology, raw data, and reproduction instructions: BENCHMARK.md. Interactive charts: browser bench.

Stream, then seal

Batch compilers are not the only path. Producers can emit aligned .nxb bytes while a segment is still open — readers parse complete records as they arrive and only need the footer tail pointer once the writer seals the file.

  • Streamable .nxb Preamble TailPtr = 0 during ingest; schema and data sectors append incrementally; sealing writes FooterTailPtr + MagicFooter at EOF.
  • Append-only WAL (.nxsw) Hot paths append NYXO rows without rewriting the tail-index every span; seal replays into a fully indexed .nxb segment.
  • Incremental readers MIT drivers expose stream parsers (e.g. NxsStreamReader) so browsers and agents can consume records before the download finishes.

How it works

NXS moves parsing cost from query time to compile time. Producers emit structured .nxb payloads; warehouses and dashboards read by pointer, not by re-parsing JSON at scale.

                                                        │
Tableau / BI ◄── Apache Arrow bridge ◄── Snowflake ingest (pointer-speed reads)

The open-core compiler and CLI compile human-readable sources; MIT SDKs embed in services without copyleft friction. Enterprise extensions add the in-memory compactor and zero-copy Arrow bridge for multi-terabyte pipelines.

Built for enterprise scale

Nyxis Core and multi-language SDKs are free and open for production within BSL limits — backed by closed-source Enterprise Extensions for multi-terabyte data grids.

  • Nyxis In-Memory Compactor Daemon (nxs-compactd) Automated background defragmentation that reclaims data-sector slack from append-heavy WAL segments, atomically swapping compacted files without racing active writers.
  • Apache Arrow zero-copy memory bridge High-throughput analytical projection via the Arrow C Data Interface — stream Nyxis payloads into Polars, DuckDB, Snowflake, and Tableau at register speed.
  • Distributed schema registry & proxy (nxs-registryd) Centralized gRPC contract management with payload validation and additive-only drift resolution (MVP).
  • Platform operations (Tier 4 v1) Encrypt-at-rest (nyxis-encrypt), file:// segment replication (nxs-replicate), and read-only query (nxs-queryd with /metrics) — Enterprise Core.
