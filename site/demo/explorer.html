What this page tests UI scalability for a very large log-shaped dataset (millions of rows): a fixed row pool and a tall inner spacer with scroll scaling map viewport position to record indices without creating one DOM node per line. With .nxb, loads use NxsStreamReader so schema and records parse while bytes arrive (v1.1 streamable files); after the footer seals, NxsReader tail-index enables random access without a full JS object graph. With .json, the file is parsed once into an array, then the same virtual list runs over that in-memory model (useful for comparison). Search, jump-to-line, and the status bar’s render timing show whether scrolling stays smooth under load.
