Live ticker

A 60 FPS rAF loop updates record 0's score every frame on both paths. JSON re-parses the whole array every K frames (simulating a server push). NXS patches 8 bytes in place.

What this page tests Main-thread frame budgeting under a continuous requestAnimationFrame loop. Both columns update the same logical field (record[0].score) each frame; pressure repeats that work multiple times per frame. The JSON column mutates a parsed array and, every K frames, JSON.stringifys and JSON.parses the full ~15 MB document (simulating a replaced server payload). The NXS column writes via DataView.setFloat64 at a cached byte offset into mapped .nxb bytes — no per-frame allocations. The stats and sparklines show FPS, per-path work time, and frames over a 20 ms budget; the Long tasks panel lists browser longtask entries (>50 ms), which usually spike on the JSON path when re-parse aligns with rAF.
