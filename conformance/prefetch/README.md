# Prefetch conformance

Vectors and runners for adaptive prefetch per `Adaptive-prefetch-spec.md` §9.4.

| Test | Phase | Gate |
|------|-------|------|
| `prefetch_viewport_basic` | 1 | required |
| `prefetch_range_coalescing` | 1 | required |
| `prefetch_sequential_upgrade` | 2 | required |
| `prefetch_cancel` | 3 | required |
| `prefetch_memory_eviction` | 1 | required |
| `prefetch_columnar_fast_path` | 4 | required |
| `prefetch_deduplication` | 1 | required |

Harness contract: inject a **fetch recorder** (HTTP range or `pread`/`ReadAt`) and assert call counts and byte ranges.

## Fixtures

| File | Description |
|------|-------------|
| `prefetch_sparse_50.nxb` | 55 row-layout records; large `tag` blobs every 10 records spread tail-index offsets across multiple 64 KiB pages |
| `prefetch_sparse_50.expected.json` | Minimal decode assertion (`id` per record; `tag` omitted) |
| `prefetch_sequential_upgrade.nxb` | 200 compact row records (`id`, `tag`); file &lt; 10 MB for eager upgrade |
| `prefetch_sequential_upgrade.expected.json` | Decode all 200 records; assert `prefetch.strategy == "eager"` and `pattern == "sequential"` after 150 sequential `record()` calls |

Generated by `make conformance-generate` into this directory.

### `prefetch_sequential_upgrade` (phase 2)

1. Open the fixture with default adaptive prefetch (no hint).
2. Call `record(i)` for `i = 0..149` sequentially.
3. Call `warmup()` to drain eager background work.
4. Assert `cache_stats().strategy == "eager"`, `pattern == "sequential"`, and `fetches_issued >= 1`.

Rust driver: `cargo test -p nxs prefetch_sequential_upgrade_conformance_vector` (skips if fixture missing).

## Runners

| Script / package | Status |
|------------------|--------|
| `harness/` (Go) | `CoalescePageIndices`, `ByteRanges`, `TestSpeculativeNonBlocking` — `go test ./...` |
| `run_prefetch_js.sh` | JS unit tests + stub vector runner when `prefetch.js` exists |

```bash
# Harness only (requires nyxis-drivers sibling checkout)
cd conformance/prefetch/harness && go test ./...

# Full prefetch gate (requires PREFETCH=1)
make conformance-prefetch PREFETCH=1
```

When the JS driver is ready, `run_prefetch_js.sh` invokes Node with a fetch recorder against `prefetch_sparse_50.nxb` for phase 1 vectors and against `prefetch_sequential_upgrade.nxb` for phase 2 sequential upgrade.
